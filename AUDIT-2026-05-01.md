# act.qcpaintshop.com — Upgrade Prompt (generated 2026-05-01)

## 0. Executive Summary

This is a feature-rich, ~70k LOC platform whose **product surface has run far ahead of its UI/CSS/security hygiene**. Three findings dominate: (1) **104 of 106 HTML pages still load Tailwind from `cdn.tailwindcss.com`** — the JIT pipeline (`npm run build:css`) was wired up but only the two password pages were migrated, so every production page ships ~3 MB of unminified runtime CSS and breaks CSP; (2) **89 of 97 migrations are unapplied** on this checkout per `migrate.js --status`, indicating either a parallel hand-written DB or a broken migration tracker — many "shipped" features (collections, AI tables, painter premium, anomalies, geo-enforcement) all show PENDING; (3) **OTP generation, the customer-portal SMS endpoint, and SMS transport are all weak** — `Math.random()` for 6-digit codes, no `otpLimiter` middleware on `/api/customer/auth/send-otp` (only inline DB count), and the SMS gateway is called over plaintext `http://` with the gateway password in the query string.

Highest-leverage themes: **(a)** finish the Tailwind JIT migration + tighten CSP and ship a real `helmet` config; **(b)** harden the OTP/SMS stack and apply the missing migrations behind a clean baseline; **(c)** consolidate the ~9 levels of inline-styled green/gold/purple gradients into the existing `design-system.css` tokens (currently every staff page hand-rolls `linear-gradient(135deg, #1B5E3B …)` inline).

## 1. Design / UX Upgrades

### 1A. Tailwind CDN → JIT pipeline (P0, blanket)

Only `forgot-password.html` and `reset-password.html` use `/css/tailwind.css`. **All other 104 pages** still ship `<script src="https://cdn.tailwindcss.com"></script>`. Findings:

- `tailwind.config.js` already scans `./public/**/*.{html,js}` and has the brand colours (`qcgreen`, `qcgold`, `admin.primary`) defined, plus a `safelist` covering JS-string interpolation. The pipeline is ready — just nothing consumes it.
- `src/tailwind-input.css` exists. Runbook: `npm run build:css` produces `public/css/tailwind.css`, which is what the two password pages already include.
- Pages confirmed still on CDN (sample): `index.html` (line 14), `login.html`, `admin-dashboard.html` (line 14), `staff/dashboard.html` (line 17), `admin-painters.html`, `admin-zoho-dashboard.html`, every `admin-*`, `customer-*`, `painter-*`, all 13 `staff/*.html`. Full list is the 104 files in the audit.
- Side effects of CDN: (i) blocks any reasonable Content-Security-Policy; (ii) inflates first-paint by ~3 MB of unminified utilities; (iii) the JIT `safelist` regex in `tailwind.config.js` is **never used today** because no page reads `tailwind.css`.

**Fix**: Replace `<script src="https://cdn.tailwindcss.com"></script>` with `<link rel="stylesheet" href="/css/tailwind.css">` on each page. Add `npm run build:css` to PM2 deploy. (P0)

### 1B. Inline styles vs. design-system tokens (P1)

`public/css/design-system.css` (1309 lines) defines `--color-primary: #6366f1` etc. but **the staff/painter pages override the token system with inline `linear-gradient(135deg, #1B5E3B…)` literals on every component**. Examples in `public/staff/dashboard.html`:

- L128: `background: linear-gradient(135deg, #1B5E3B15, #154D3115);`
- L168: `background: linear-gradient(135deg, #1B5E3B 0%, #154D31 100%);`
- L474: `background: linear-gradient(90deg, #1B5E3B, #154D31);`
- L653/L697/L704/L712: every action button has `style="padding: 12px 8px; border-radius: 12px; font-size: 13px"` instead of using `.action-btn` from design-system.

**Fix**: Add staff/painter palette to `:root` in `design-system.css` (`--color-staff-primary: #1B5E3B`, `--color-staff-dark: #154D31`, `--color-painter-gold: #D4A24E`) and the gradients (`--gradient-staff`, `--gradient-painter`). Replace inline `style="background: linear-gradient(...)"` with class names. ~30 staff/painter pages. (P1)

### 1C. Mobile / tap-target audit (P0/P1)

- **`public/staff/dashboard.html` L553** — notification bell is `width:40px;height:40px` (Apple HIG minimum is 44×44). Bump to 44/44. (P0)
- **`public/staff/dashboard.html` L697-712** — break/outside-work/prayer mini buttons stack at `padding: 12px 8px; font-size: 13px;` with two-line vertical labels. On 375px iPhone SE these clip below the gradient header. Increase to `padding: 14px 10px` and reduce label to one word. (P1)
- **13 staff pages** set `user-scalable=no` (`maximum-scale=1.0, user-scalable=no`) which is an a11y violation per WCAG 1.4.4. Drop `user-scalable=no` everywhere. (P1)
- **`public/admin-painters.html` L1669-1688** — admin painters table renders 9 columns on mobile with no horizontal-scroll wrapper. Wrap `<table>` in `<div class="overflow-x-auto">`. (P1)
- **`public/admin-zoho-dashboard.html`** — 7-card KPI grid uses `grid-cols-2 md:grid-cols-4 lg:grid-cols-7`; on tablets produces 7 cramped cards. Use `lg:grid-cols-4 xl:grid-cols-7`. (P2)

### 1D. Accessibility (P1)

- **Zero `aria-label` / `role` attributes** on `public/staff/dashboard.html` (grep returns 0). Same for many admin pages.
- Bell icon, profile avatar, hamburger and the four mini activity buttons all use SVGs with no accessible text. Add `aria-label` to every icon-only button. (P1)
- Many `<button>`s use `style="background: linear-gradient(...);"` with white text; some pairs (e.g. `#D4A24E` gold on white in painter dashboard) fail WCAG AA contrast. Run a Lighthouse contrast pass. (P1)
- `public/index.html` ships `<html lang="en">` but mixes Tamil text. Wrap Tamil spans in `<span lang="ta">` for screen-reader correctness. (P2)

### 1E. Visual consistency / theme (P1)

- **Brand mismatch**: `design-system.css` `:root` is purple-indigo (admin), but staff & painter pages override with inline green. Consider a body-class strategy: `<body class="theme-staff">` / `theme-painter` / default admin, with each theme overriding the same tokens.
- `public/index.html` L26: `nav-cta` uses admin gradient `#667eea→#764ba2`. The public landing page should be brand-neutral or match the dominant green. (P2)
- **Loading skeletons**: `css/skeletons.css` and `js/ui-skeletons.js` exist (D5 shipped) but only ~20 of 106 pages call them. Wire `qcSkeleton.show(container, 'cards')` into `loadLeads()` on `staff-leads.html`. (P1)

### 1F. Customer journey issues (P1)

- **`public/customer-dashboard.html`** — customers viewing estimates on a phone get a horizontally-scrolling table. Add a card layout below `md:` breakpoint. (P1)
- **`public/share/estimate.html`** — missing OG meta tags (`og:image`, `og:title`). WhatsApp shares show no preview. (P1)
- **`public/customer-login.html`** — OTP entry 6 inputs lack `inputmode="numeric"`, so iOS opens alpha keyboard. Add `inputmode="numeric" pattern="[0-9]*"`. (P1)
- **`public/login.html`** — no "Show password" toggle. (P2)

### 1G. Print / PDF (P2)

## 3. Reliability / Bug / Security Fixes

### CRITICAL

- **`server.js:785, 960, 3183`, `routes/painters.js:265`** — OTP uses `Math.random()`. Brute-forceable in ~50k requests with timing. **Fix**: `crypto.randomInt(100000, 1000000)`.
- **`server.js:813, 983, 3211`, `routes/painters.js:294`** — SMS gateway called over `http://` with gateway `password=` in query string. OTP visible in network logs. **Fix**: switch to HTTPS; send credentials in POST body.
- **`routes/painters.js:264-265`** — Hard-coded test bypass: `phone === '9999999999'` returns OTP `'123456'`. Gate on `process.env.NODE_ENV !== 'production'` or remove entirely.
- **`server.js:3146`** — `POST /api/customer/auth/send-otp` has no `otpLimiter` middleware — only inline DB count, no atomic check. Two parallel requests can both pass. Add `otpLimiter` middleware.
- **`server.js:1077-1086`** — `/api/auth/register` checks existing by phone but not username — collision possible with admin usernames. Add username uniqueness check.
- **`routes/painters.js:144-160` `requirePainterAuth`** — `painter_sessions.token` matched against raw header (no hashing). DB leak = session takeover. Hash token at rest.

### HIGH

- **`server.js:415`** — `user_sessions` stores `session_token` in **plaintext**. DB leak = every active staff session hijackable. Hash on insert; compare hash on `requireAuth`.
- **`server.js:135`** — `if (!origin) return callback(null, true)` allows requests with no `Origin` header — tightens CSRF surface.
- **`server.js:594`** — `forgot-password` timing oracle: SMTP send creates noticeable latency delta between known/unknown emails. Move `sendMail` into `setImmediate()` after `res.json()`.
- **`routes/share.js:49`** — dynamic `tableName` pattern dangerous; future contributors may mis-validate. Use a `Map` of fixed strings.
- **Multiple routes** — dynamic `UPDATE … SET ${setClauses}` where setClauses columns are derived from `req.body` keys. An attacker can target any column they can guess (e.g. `password_hash`). Add `ALLOWED_FIELDS = new Set([...])` per route. Key files: `routes/system.js:599,697`, `routes/wa-marketing.js:210,567`, `routes/zoho.js:2665,3038,4621`, `routes/painters.js:578,4247,5803,6255`, `routes/estimates.js:977`, `routes/roles.js:278`.

### MED

- **`server.js:117-157`** — CORS falls back to a default if `CORS_ORIGIN` unset with a `console.error`. Should `process.exit(1)` in production.
- **`server.js:103-112`** — error buffer monkey-patch doesn't cap the `stack` field. Memory growth risk. Cap to 4 KB.
- **89 PENDING migrations** — run `node migrate.js --mark-existing` after verifying tables exist.
- **`server.js:1715`** — `GET /api/ai-status` calls `model.generateContent('Say OK')` on every request — burns Gemini quota for liveness. Cache result 60 s.
- **Dead/backup files** — `server.js.bak.dpl` (175 KB), `middleware/permissionMiddleware.js.OLD`, `middleware/permissionMiddleware.js.backup`, `routes/attendance.js.OLD`, `routes/attendance.js.backup`, `public/dashboard.html.broken-backup`, `public/estimate-create.html.old`, `public/universal-nav-loader-backup.js`, `public/universal-nav-loader-v3.js`. Delete.
- **`routes/painters.js` is 6316 lines**, `routes/zoho.js` is 4839, `server.js` is 4105. Split into sub-route files.
- **`public/admin-painters.html:1582-1584,1943`** — error paths set `innerHTML` with raw `err.message`. If message contains HTML it will render. Use existing `esc()` helper.
- **No `audit_records` purge job** — table will grow unbounded. Add a 90-day TTL cron.
- **`services/customer-auth.js`** hashes customer tokens (good). `user_sessions` does not — inconsistent. Align.

### LOW

- **`server.js:411`** — admin session 720 h (30 d). Cap admin sessions at 7 d.
- **`config/uploads.js`** — upload filenames use `Math.random()*1E9` — enumerable. Use `crypto.randomBytes(8).toString('hex')`.
- **`routes/share.js:106`** — 30-day public share link with no revocation UI. Add admin "Revoke link" button.
- **`public/admin-painters.html:1670`** — per-row `onclick` on 50-row table. Use event delegation.
- **`public/sw.js`** — service worker only caches `OFFLINE_URL`. Add a small precache list for CSS/JS assets.
- **`server.js:419, 469, 519`** — three near-identical session lookups. Extract to a helper.

## How to Use

1. **PRD** defines the full product scope, personas, and success metrics
2. **User Stories** are organized by epic with acceptance criteria and priority
3. **Architecture** documents the current system + proposed improvements
4. Stories move from Backlog → Sprint → In Progress → Done
5. Each implementation story references the architecture doc for design decisions

## BMAD Personas Active

- **Analyst**: Requirements gathering, gap analysis
- **Product Manager**: PRD ownership, prioritization
- **Architect**: System design, technical strategy
- **Scrum Master**: Story breakdown, sprint planning
